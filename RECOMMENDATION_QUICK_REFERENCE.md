Troubleshooting\n\n### Issue: Slow recommendations (>1000ms)\n\n**Solution 1: Check database indexes**\n```bash\n# MongoDB\ndb.products.createIndex({ \"category\": 1, \"stock\": 1 })\ndb.products.createIndex({ \"rating\": -1, \"numReviews\": -1 })\n```\n\n**Solution 2: Reduce candidate pool**\n```javascript\nRECOMMENDATIONS.CANDIDATE_POOL = 50; // was 240\n```\n\n**Solution 3: Increase cache TTL**\n```javascript\nCACHE.TTL_SECONDS = 10 * 60; // was 5\n```\n\n### Issue: Recommendations not personalized\n\n**Check user interests:**\n```bash\n# MongoDB\ndb.users.findOne({ _id: userId }, { interests: 1 })\n```\n\n**Verify search tracking:**\n```bash\n# Check logs\nLOG_LEVEL=debug npm run dev\n# Look for: \"Search intent tracked\" or \"Search Intent Captured\"\n```\n\n**Check cache invalidation:**\n```bash\n# Redis\nredis-cli\n> KEYS recommendations:v3:*\n> TTL recommendations:v3:userId123\n```\n\n### Issue: High error rate\n\n**Check metrics:**\n```javascript\n// In browser console\nconst res = await fetch('/api/user/recommendation-metrics', {\n  headers: { 'Authorization': 'Bearer {token}' }\n});\nconsole.log(await res.json());\n```\n\n**Enable debug logging:**\n```bash\nLOG_LEVEL=debug npm run dev\n```\n\n**Check database connection:**\n```bash\n# Test MongoDB\nmongo --eval \"db.adminCommand('ping')\"\n\n# Test Redis\nredis-cli ping\n```\n\n### Issue: Frontend retry loop (retrying endlessly)\n\n**Check network:**\n- Is server running?\n- Is port correct in `.env`?\n\n**Check token:**\n- Is token valid?\n- Is it being sent in Authorization header?\n\n**Check logs:**\n```bash\nLOG_LEVEL=debug npm run dev\n```\n\n## Monitoring Dashboard (DIY)\n\n### Monitor Engine Health\n\n```javascript\n// Create dashboard.html\n<!DOCTYPE
